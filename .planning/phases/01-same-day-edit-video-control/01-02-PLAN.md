<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| guest (share token) → public video-status | Unauthenticated; must not learn a review cut exists or get its videoUrl before approval. |
| host (Clerk) → approve/regenerate | Authenticated but must be the event owner; mutating endpoints with side effects (notifications). |
| worker → DB | Worker writes job terminal state; must not trigger any notification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | public token video-status | mitigate | If `approvedAt` is null, return benign in-progress shape with `videoUrl: null`; only expose videoUrl when approvedAt set. Asserted by test. |
| T-02-02 | Elevation of Privilege | approve/regenerate | mitigate | `requireAuth` + `event.hostId !== user.id -> 404` (ownership returns 404, not 403, to avoid existence leak). Asserted by test. |
| T-02-03 | Tampering / abuse of trust | premature/double notification | mitigate | Fan-out removed from worker and moved to approve; approve idempotent on `approvedAt` (no re-notify). Asserted by test. |
| T-02-04 | Tampering | Express 5 req.params into eq() | mitigate | `String(req.params.eventId)` on every new handler (copied from /end). |
| T-02-05 | Denial of Service | regenerate while job processing | accept | Marking current job supersededAt and enqueuing a fresh pending job is bounded; single in-process worker drains one job; low-value target, no amplification. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages; no install task. |
</threat_model>
